# META FICTION SYSTEM — THE ARCHITECT'S WORKBENCH

Welcome to the **Meta System**. This is the control room where the Ultimate AI Fiction System is designed, refined, and upgraded. 

You are **not** here to write a book. You are here to build the machine that writes the books.

## The Goal

To maintain and endlessly improve the `_Template_Fiction_System/`—a world-class, multi-agent framework capable of generating masterpiece-level novels, epic series, screenplays, and multi-modal assets (images, concept art).

## Directory Purpose

- **`_Meta_Fiction_System/`**: This directory. It contains the rules, charters, and workflows for the AI agents acting as System Architects. 
- **`_Template_Fiction_System/`**: The payload. This is the master template that gets copied when a human author wants to start a new project.

## The Meta Philosophy

1. **Systemic Over Ad-Hoc:** If a book project runs into a recurring issue (e.g., agents forgetting a side character), we do not fix it in the book. We fix it *here* by designing a better `canon/` ingestion protocol in the template.
2. **Agent-Agnostic Design:** The template must work seamlessly whether the human is using Claude via web interface, Gemini via API, Cursor IDE, or Agent Zero on the command line.
3. **Relentless Pursuit of Quality:** We do not settle for "good enough" AI text. The prompts and rules we engineer here must squeeze absolute literary brilliance out of the models.

To begin work, review `OPEN_QUESTIONS.md` or consult the human for the next architectural upgrade.
